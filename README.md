# Java 2048 Logic Engine

## 项目简介
本项目是一个基于 Java 实现的 2048 游戏核心逻辑引擎。该项目将游戏业务逻辑与图形渲染层进行解耦，核心开发重点在于处理 4x4 矩阵在不同方向倾斜（Tilt）时的数字合并算法与状态转换。

## 技术选型
* 核心语言：Java 1.8+
* 测试框架：JUnit 4
* **外部依赖**：`library-sp24/` (包含核心渲染库、JUnit 测试组件及其他必要的第三方依赖)
* 开发模式：测试驱动开发 (TDD)
* 架构模式：MVC (模型-视图-控制器) 逻辑分离设计

---

## 技术亮点与实现思路

### 1. 矩阵滑动合并算法优化

在处理游戏核心的滑动逻辑（Tilt）时，本项目采用了抽象化的处理策略：


* 方向归一化：利用矩阵旋转思想，将四个方向的滑动合并操作统一抽象为“向上滑动”逻辑。通过坐标变换，避免了编写大量重复的条件判断语句，显著降低了代码冗余度。

* 合并规则约束：严格执行单次移动仅限一次合并的规则。通过双指针扫描列向量，在 O(N) 时间复杂度内完成原地（In-place）的数据压缩与合并。

### 2. 高覆盖率单元测试 (TDD 实践)

本项目高度重视逻辑的正确性与稳健性，开发全过程遵循测试驱动开发流程：

* 场景覆盖：编写并运行了包括单列合并、多列并行滑动、极限满屏状态、胜负临界状态在内的 10 余项专项测试集。

* 质量保障：确保核心模块 `game2048logic` 的代码逻辑通过全量自动化测试，具备极高的鲁棒性。

### 3. 解耦架构设计

* 逻辑层 (game2048logic)：封装所有与规则相关的状态机，不依赖任何 GUI 组件，可轻松移植至 Web 服务端。

* 渲染层 (game2048rendering)：负责图形化展示与用户按键事件响应，通过清晰的接口与逻辑层交互。
